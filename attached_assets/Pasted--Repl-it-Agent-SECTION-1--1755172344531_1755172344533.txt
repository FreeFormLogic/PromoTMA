1.1. Роль Агента и Основная Директива

Ты — эксперт full-stack разработчик, специализирующийся на создании отзывчивых, высокопроизводительных веб-приложений с использованием Vue.js и Tailwind CSS. Ты пишешь чистый, компонентный и хорошо документированный код.
Твоя основная задача — создать одностраничный веб-компонент с именем AdvantagesPage.vue. Эта страница будет служить комплексной базой знаний под названием "Ваше преимущество" в рамках существующего Telegram Mini App. Страница должна быть интерактивной, визуально привлекательной и структурированной для обслуживания двух разных аудиторий через интерфейс с вкладками.
Ключевая цель этой страницы — не просто информировать, а убеждать и вооружать. Вкладка "Для Клиента" — это прямой коммерческий питч, переводящий функции в ощутимую бизнес-ценность. Вкладка "Для Продавца" — это "боевая карта", предоставляющая данные и аргументы, необходимые для победы в переговорах. Структура должна отражать эту двойственность.

Фреймворк: Vue.js 3 (Composition API).
Стилизация: Tailwind CSS. Убедись, что все стили написаны по принципу utility-first и полностью адаптивны.
Иконки: Используй библиотеку heroicons для всех иконок, упомянутых в дизайне. Для этого импортируй необходимые иконки (например, CheckCircleIcon, ArrowRightIcon, PlusIcon, ChevronDownIcon) из @heroicons/vue/24/solid или @heroicons/vue/24/outline.
Управление состоянием: Компонент должен быть самодостаточным. Управляй его локальным состоянием (например, активной вкладкой) с помощью ref или reactive из Vue.

2.2. Компонент Навигации по Вкладкам

Реализуй чистую, современную панель навигации по вкладкам.
Используй ref для отслеживания активной вкладки, например: const activeTab = ref('client');.
Создай две кнопки или ссылки для переключения вкладок.
Вкладка 1: "Преимущества для Клиента". Эта вкладка должна быть активной по умолчанию.
Вкладка 2: "Аргументы для Продавца".
Активная вкладка должна иметь отчетливый визуальный стиль. Например, border-blue-500 text-white. Неактивная вкладка: border-transparent text-gray-400 hover:text-white.
Пример реализации:

HTML


<div class="border-b border-gray-700 mb-8">
  <nav class="-mb-px flex space-x-8" aria-label="Tabs">
    <button @click="activeTab = 'client'"
            :class="">
      Преимущества для Клиента
    </button>
    <button @click="activeTab = 'sales'"
            :class="">
      Аргументы для Продавца
    </button>
  </nav>
</div>



2.3. Панели с Контентом

Создай два основных контейнера div. Используй v-if для отображения контента, соответствующего активной вкладке. Это обеспечивает лучшую производительность, так как неактивный контент не будет находиться в DOM.

HTML


<div>
  <div v-if="activeTab === 'client'">
    </div>
  <div v-if="activeTab === 'sales'">
    </div>
</div>


Интерфейс с вкладками является критически важным выбором для UX. Он предотвращает информационную перегрузку и позволяет каждому типу пользователей немедленно получить доступ к релевантному контенту, не прокручивая ненужные данные. Это снижает когнитивную нагрузку и делает инструмент гораздо более эффективным, решая проблему длинного, неструктурированного списка преимуществ.
